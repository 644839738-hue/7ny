## 完整脚本

### 段落 1：项目介绍（30 秒）

```
[画面：浏览器打开 SpriteForge AI 首页 Dashboard]

今天给大家演示 SpriteForge AI ——
一个面向 2D 游戏开发的素材生成与资产化管线工具。

假设你正在做一款叫 "Dark Dungeon" 的像素风 Roguelike 游戏，
你需要角色、怪物、地牢墙壁 Tile、还有 UI 血条，
但你没有美术——这个工具就是为你准备的。

核心区别：普通的 AI 生图工具产出的是"图片"，
SpriteForge AI 产出的是"游戏引擎可直接导入的资产"。
```

**关键画面**：
- Dashboard 页面总览
- 侧边栏导航（生成 / Sprite Sheet / Tile / 导出）

---

### 段落 2：创建项目 & 设定主题（25 秒）

```
[画面：切换到 Settings 页面]

我们先用 Settings 创建一个项目配置。

项目名称：Dark Dungeon
目标引擎：Godot（因为我们习惯用 Godot 做 2D 游戏）

这个配置会被导出步骤使用——确保 ZIP 包的目录结构
和 Godot 项目的 assets/sprites/ 约定一致。
```

**关键画面**：
- Settings 页面填写 projectName
- 选择 Godot 引擎

---

### 段落 3：生成素材（50 秒）

```
[画面：切换到素材生成页面]

现在开始生成素材。先说角色：

素材类型选 Character（角色），
Prompt 写 "dark dungeon knight with sword, pixel art"，
风格选 Pixel Art，尺寸 32×32，每次生成 4 张，
背景透明。

点生成。Demo 模式下使用内置样例素材，
所以你看到的结果是即时返回的。

[等待生成完成，展示结果]

接下来我们换 Tile 类型，Prompt 写 "dark stone dungeon floor tile"，
生成 4 张地牢地板砖。

然后是 Item（道具），Prompt 写 "glowing health potion"，
做一瓶发光的血药。

最后是 UI 元素：Prompt 写 "dark fantasy health bar border"。
```

**关键画面**：
- Character 生成结果
- Tile 生成结果
- Item 生成结果
- UI 生成结果
- Demo 标记可见

**解说强调**：无外部 API，全 Demo 模式跑通。

---

### 段落 4：后处理管线（30 秒）

```
[画面：终端显示后端日志，或浏览器显示素材处理后的效果]

生成完成后，后端自动执行后处理管线：

1. 透明背景移除 —— 确保素材是 RGBA PNG
2. 空白区域裁剪 —— 去掉多余的透明边距
3. 尺寸标准化 —— 缩放到目标尺寸（32×32）
4. 主体居中 —— 素材在画布正中

这些步骤全部自动完成，你不需要打开 Photoshop。
```

**关键画面**：
- 处理前后的素材对比（如果有条件展示）
- 或展示后端 output 目录中的处理后文件

---

### 段落 5：Sprite Sheet 拼接与动画预览（35 秒）

```
[画面：切换到 Sprite Sheet 页面]

有了素材帧，我们来做 Sprite Sheet。

步骤 1：输入刚才的 task_id 加载角色帧。
步骤 2：设置动画参数：
  - 动画名：knight_walk
  - FPS：12
  - 帧宽/帧高：32×32
  - 列数：4

点击生成。右边是拼接好的 Sprite Sheet 图——
注意背景是棋盘格，说明透明通道完好。

下面是动画预览——可以看到角色在播放 walk 动画。
CSS 直接通过 backgroundPosition 切帧，不需要任何第三方库。

下面是帧列表和 JSON 元数据，
记录了每帧的坐标、尺寸、动画范围。
```

**关键画面**：
- Sprite Sheet 拼接图（带棋盘格背景）
- 动画预览播放中（帧指示器跳动）
- JSON 元数据展开
- 下载 PNG / JSON 按钮

---

### 段落 6：Tile 平铺预览与边缘评分（40 秒）

```
[画面：切换到 Tile 预览页面]

Tile 素材生成后，最关键的问题是——
它平铺后边缘会不会断裂？

我们来检查。输入 task_id 加载 tile 素材，
选择一张地牢地砖，点击「生成平铺预览并评分」。

左边是 3×3 平铺效果——一眼就能看出重复排列的效果。
右边是边缘一致性评分：

当前 95 分，Excellent（优秀），
上-下边一致性 96 分，左-右边一致性 94 分。
下方有建议：适合平铺，边缘过渡自然。

这个评分是基于四边 RGB 均值欧几里得距离计算的——
完全自研算法，不需要任何外部服务。
```

**关键画面**：
- 3×3 平铺预览图
- 评分仪表盘（数字 + 进度条 + 评级）
- 建议文案

---

### 段落 7：导出引擎 ZIP 包（30 秒）

```
[画面：切换到导出页面]

所有素材都准备好了，最后一步——导出。

输入 task_id 加载要导出的素材，
选择目标引擎 Godot，
勾选「包含 Sprite Sheet」「包含 Tile 平铺预览」。

点击导出。

导出成功！9 个文件，12KB。
下载 ZIP 并解压看看里面有什么：

  images/            → 4 张独立素材
  spritesheets/      → Spritesheet PNG + JSON 帧元数据
  tiles/             → Tile 3×3 预览图
  manifest.json      → 完整素材清单
  README_IMPORT.md   → Godot 引擎导入说明

README 里面有详细的导入步骤——
Filter 取消勾选、Lossless 压缩、AnimationPlayer 设置等。
```

**关键画面**：
- 导出页面三个步骤
- 下载按钮
- ZIP 解压后的目录结构（终端 tree 命令或文件浏览器）
- README_IMPORT.md 内容特写
